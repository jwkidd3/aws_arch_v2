AWS Storage Services Overview

Three Primary Storage Types

  • S3 (Simple Storage Service): Object-based storage
  • EBS (Elastic Block Store): Block-based storage for EC2
  • EFS (Elastic File System): Network file system

Storage Services Comparison

Service Type Use Case
S3 Object Storage Web hosting, backups, archives
EBS Block Storage OS, databases, file systems
EFS File Storage Shared access, Linux only

Amazon S3 Deep Dive

S3 Core Features

  • Durability: 99.999999999% (11 9s) reliability
  • Global Namespace: Unique bucket names worldwide
  • Regional Resource: Data stays in chosen region

S3 Key Capabilities

Essential Features

  • API Access: REST API for all operations
  • Web Interface: AWS Console management
  • WORM Compliance: Write Once, Read Many support
  • Versioning: Multiple versions of objects

S3 Storage Classes

Class Use Case Availability
Standard Frequently accessed data 99.99%
Standard-IA Infrequently accessed 99.9%
Glacier Long-term archival 99.99%

S3 Static Website Hosting

Website Hosting Features

  • HTML/CSS/JS: Serve static web content
  • Custom Domain: Use your own domain name
  • Error Pages: Custom 404 and error handling
Note: S3 website endpoints differ from bucket endpoints

Amazon Elastic Block Store (EBS)

EBS Fundamentals

  • Block-Level Storage: Raw block storage for EC2
  • Persistent: Data persists beyond instance lifecycle
  • AZ-Specific: Volumes exist within single AZ

EBS Volume Types

Type Use Case IOPS
gp3 General purpose 3,000-16,000
io1/io2 High IOPS Up to 64,000
st1 Throughput optimized N/A

EBS Volume Types - Details

Volume Characteristics

  • gp3: Latest generation, cost-effective balanced performance
  • io1/io2: Provisioned IOPS for critical applications
  • st1: Throughput-optimized for big data workloads
  • sc1: Cold storage for infrequently accessed data

EBS Snapshots

Snapshot Features

  • Point-in-Time Backup: Incremental backups to S3
  • Cross-Region Copy: Disaster recovery capability
  • Volume Creation: Create new volumes from snapshots

EBS Snapshot Best Practices

Backup Strategy

  • Regular Schedule: Automated snapshot creation
  • Application Consistency: Stop I/O before snapshot
  • Lifecycle Management: Delete old snapshots automatically

EBS Encryption

Security Features

  • Encryption at Rest: Data encrypted on disk
  • Encryption in Transit: Data encrypted in motion
  • Key Management: AWS KMS integration
  • Snapshot Encryption: Automatic snapshot encryption

EBS Performance Optimization

Performance Factors

  • Instance Type: EBS-optimized instances
  • Volume Size: Larger volumes = better performance
  • IOPS Provisioning: Match workload requirements

AWS Database Services Overview

Managed vs Unmanaged

  • Unmanaged: You manage scaling, fault tolerance, availability
  • Managed: AWS handles infrastructure, scaling, backups

Relational vs Non-Relational Databases

Aspect Relational Non-Relational
Structure Rows and Columns Key-Value, Document
Schema Fixed Schema Dynamic Schema
Query SQL Document/API Based

Database Scaling Patterns

Type Scaling ACID
Relational Vertical (Scale Up) Full ACID Support
Non-Relational Horizontal (Scale Out) Eventual Consistency

AWS Database Service Portfolio

Primary Database Services

  • Amazon RDS: Managed relational databases
  • Amazon Aurora: Cloud-native relational database
  • Amazon DynamoDB: Managed NoSQL database
  • Amazon ElastiCache: In-memory caching

Amazon RDS Deep Dive

RDS Supported Engines

  • MySQL: Popular open-source database
  • PostgreSQL: Advanced open-source database
  • Oracle: Enterprise database system
  • SQL Server: Microsoft database platform

RDS Deployment Options

Single-AZ vs Multi-AZ

Single-AZ: One instance | Multi-AZ: Primary + Standby

Multi-AZ Benefits

  • High Availability: Automatic failover
  • Synchronous Replication: Data consistency

RDS Multi-AZ Configuration

How Multi-AZ Works

  • Primary Instance: Serves read/write traffic
  • Standby Instance: Synchronously replicated backup
  • Automatic Failover: DNS endpoint switches automatically
Failover Time: Typically 60-120 seconds

RDS Read Replicas

Read Replica Features

  • Read Scaling: Offload read traffic from primary
  • Cross-Region: Deploy replicas in different regions
  • Asynchronous: Eventually consistent replication
  • Promotion: Can be promoted to standalone DB

RDS Backup & Recovery

Backup Types

  • Automatic Backups: Point-in-time recovery (7-35 days)
  • Manual Snapshots: User-initiated, persist until deleted
  • Cross-Region: Copy snapshots to other regions

RDS Security Features

Security Controls

  • VPC Integration: Deploy in private subnets
  • Security Groups: Network-level access control
  • Encryption: At rest and in transit
  • IAM Authentication: Database user authentication

AWS Load Balancing

Load Balancer Purpose

  • Traffic Distribution: Spread load across multiple targets
  • High Availability: Eliminate single points of failure
  • Health Monitoring: Route traffic only to healthy targets

Load Balancer Architecture

Load Balancer Components

Listener â†’ Rules â†’ Target Groups â†’ Targets

Key Components

  • Listeners: Check for connection requests
  • Rules: Determine how to route requests

Load Balancer Components Detail

Target Groups & Health Checks

  • Target Groups: Logical grouping of targets
  • Health Checks: Monitor target availability
  • Targets: EC2 instances, IP addresses, Lambda functions

AWS Load Balancer Types

Type Layer Protocol
Application (ALB) Layer 7 HTTP, HTTPS
Network (NLB) Layer 4 TCP, UDP
Gateway (GWLB) Layer 3 IP

Application Load Balancer (ALB)

ALB Features

  • Layer 7 Routing: Content-based routing
  • Path-Based: Route based on URL path
  • Host-Based: Route based on hostname

ALB Advanced Features

Enhanced Capabilities

  • SSL Termination: Handle SSL/TLS encryption
  • WebSocket Support: Persistent connections
  • HTTP/2 Support: Modern protocol support
  • Lambda Integration: Serverless target support

Network Load Balancer (NLB)

NLB Characteristics

  • Ultra-Low Latency: Millions of requests per second
  • Static IP: Fixed IP addresses per AZ
  • TCP/UDP: Layer 4 load balancing
  • Extreme Performance: Handle sudden traffic spikes

Load Balancer Health Checks

Health Check Configuration

  • Protocol: HTTP, HTTPS, TCP
  • Port: Target port for health checks
  • Path: Health check endpoint
  • Thresholds: Healthy/unhealthy thresholds

SSL/TLS Termination

SSL Termination Flow

Client (HTTPS) â†’ ALB (SSL Term) â†’ Target (HTTP)

Benefits

  • Certificate Management: Centralized SSL certificates
  • Performance: Offload SSL processing from targets

Cross-Zone Load Balancing

Traffic Distribution

  • Enabled: Even distribution across all targets
  • Disabled: Distribution only within each AZ
  • ALB Default: Always enabled
